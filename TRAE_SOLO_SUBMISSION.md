# 【Code With SOLO】用 SOLO 打造跨平台多模态“第二大脑”：基于本地 AI 引擎的知识图谱 InsightGraph

## 1. 摘要：
面对信息碎片化和“记完就忘”的痛点，我使用 TRAE SOLO 在短时间内从零到一独立开发了一款全栈桌面端应用——**InsightGraph**。它深度整合了 Dify 的 AI 引擎，不仅实现了沉浸式的 Markdown 写作和极客风的块级引用，还内置了会“主动推荐”知识的 AI 图书管理员、11 种预设模板的 AI 魔法重写功能，以及极具原生感的 Apple Glass 毛玻璃主题，彻底打通了从知识摄入、管理到输出的全链路。

## 2. 背景：
我是一名开发者兼重度知识管理爱好者。在日常工作中，我需要阅读大量的 PDF 论文、RSS 资讯，并记录每日心得。然而，市面上的工具往往存在几个问题：
1. **输入摩擦力大**：PDF/网页/日记分布在不同的软件中，难以联动。
2. **知识变成死水**：存进去的资料很少再看，写新笔记时根本想不起旧资料。
3. **大模型容易产生幻觉**：直接让大模型读整篇长文档重写，效果差且 Token 消耗巨大。

因此，我希望用 SOLO 帮我快速构建一个真正“懂我”的本地第二大脑。

## 3. 实践过程：
我将整个应用的开发拆解为几个关键阶段，全程由 TRAE SOLO 深度主导和执行：

### 阶段一：全栈架构搭建与底座成型
- **任务拆解**：梳理前后端分离架构，后端使用 FastAPI + PostgreSQL，前端使用 Vue3 + Electron，大模型引擎全面委托给 Dify。
- **SOLO 能力运用**：通过 SOLO 的 **Builder 模式** 和 **RunCommand** 工具，它自动为我生成了项目脚手架、配置了 Docker Compose，甚至自己编写 `python test.py` 脚本来打通与 Dify 接口的连通性测试。

### 阶段二：AI 魔法棒与主动式知识推荐
- **任务拆解**：我需要一个 AI 写作助手，以及一个能在后台“防抖检索”历史笔记的 AI 图书管理员。
- **关键 Prompt**：“帮我开发一个基于 SSE 流式输出的 AI 魔法重写功能，支持‘模板’和‘自定义’双模驱动，提供扩写、压缩、生成大纲等 11 种预设模板。同时在侧边栏实现防抖检索，主动推荐相关文献。”
- **避坑经验**：在调试 SSE 流式响应时遇到了前端截断问题。SOLO 自动调用了终端查看 Docker 容器的报错日志，发现并修复了数据块组装的 Bug，直接交付了完美打字机特效。

### 阶段三：精准的块级高亮引用 (Block-level Reference)
- **任务拆解**：为了解决大模型幻觉，开发类似 Roam Research 的 `((` 极客引用语法。
- **SOLO 能力运用**：SOLO 精准地通过 `SearchReplace` 工具修改了大量 Vue 组件，实现了鼠标划选提取高亮、键盘拦截 `((` 呼出浮窗搜索，并将溯源链接自动插入到 Markdown 光标处。

### 阶段四：Apple Glass 毛玻璃拟物主题重构
- **任务拆解**：让软件看起来像 macOS 的原生级应用，告别枯燥的纯色。
- **SOLO 能力运用**：我触发了 SOLO 的专属技能 **`frontend-design`**。它展现了惊人的设计品味，不仅帮我把写死的 Tailwind 颜色全部重构为规范的 CSS 变量，还设计了包括科技紫、护眼森林、复古拿铁、极客纯黑以及惊艳的 **Apple Glass（果味毛玻璃）** 等五大主题。全站实现了秒级的无缝换肤。

## 5. 效果与总结：

**提效数据：**
- **知识录入**：原本需要手动复制粘贴到不同软件整理 -> 原生 PDF 划选或图片 `Ctrl+V` 一键生成胶囊，摩擦力降至 0。
- **写作输出**：阅读长文献提炼博客需要 1-2 小时 -> 极客引用 `((` 拼接高亮片段，AI 魔法重写 20 秒出稿。
- **知识重用**：旧笔记石沉大海 -> AI Librarian 打字即推，历史知识唤醒率提升 100%。
- **项目开发**：这样规模的全栈双端系统预估需要数周 -> SOLO 辅助下，极短时间内完成前后端打通与主题美化。

**SOLO 在流程中的角色：**
- **全栈代码生成**：生成 FastAPI 后端、Vue3 组件及 Electron 配置。
- **Bug 排查神兵**：遇到跨域或流式截断问题，直接贴报错或让它执行 Docker 日志读取，它能准确定位修复。
- **高级 UI 工程师**：不仅能写功能代码，在提出“Apple Glass 主题”需求时，能完美写出优雅的 CSS 变量和 `backdrop-filter` 实现商业级换肤体验。

**可复用的方法：**
- **分步拆解法**：将庞大的知识库系统拆解为：基础框架、Dify 联通测试、魔法重写、高亮引用、主题优化 5 个步骤，每步只给 SOLO 一个核心目标。
- **明确边界法**：在 Prompt 中指明“大模型极易幻觉”，引导 SOLO 为我设计“块级引用 `((`”这样的交互，而不是让 AI 去啃全文。
- **相信工具库**：鼓励 SOLO 使用内置的 Skills（如 `frontend-design`），它的设计品味往往远超预期的默认实现。
