## 2. 产品简介

InsightGraph（研见图谱）是一个基于 LangGraph 构建的研究情报操作系统，用于自动抓取 arXiv 中用户指定分类的新增论文，完成清洗、去重、版本归并、知识入库、结构化总结、分析与观点生成，并输出日报、周报、专题观察与突破追踪。

系统同时支持基于知识库的对话式精读、论文比较、背景讲解、知识链接推荐和阅读路径规划。

InsightGraph 的目标不是简单做“论文推荐”，而是构建一个从“研究发现”到“研究沉淀”再到“研究解释与追踪”的完整闭环。

---

## 3. 产品目标

### 3.1 核心目标
1. 自动发现并筛选前沿研究
2. 以结构化方式输出高价值研究情报
3. 构建可检索、可追问、可扩展的研究知识库
4. 支持围绕论文与主题持续进行精读和追踪

### 3.2 MVP 目标
1. 首期仅支持 arXiv
2. 支持配置分类与主题
3. 支持日报、周报、专题观察、突破追踪
4. 支持知识库入库和对话式精读
5. 支持 Markdown / Web / 飞书 / 企业微信输出
6. 每个分类每日最多输出 5 篇重点论文

- 元数据抓取

- 知识链接生成

## 6. 功能需求

## 6.1 数据采集模块

### 功能描述
系统定时从 arXiv 拉取指定分类的新论文元数据，并尽可能获取摘要、PDF 链接和可提取文本。

### 功能要求
1. 支持基于分类增量抓取
2. 支持按时间窗口执行抓取
3. 支持每日定时任务
4. 保存原始元数据
5. 支持失败重试
6. 支持抓取日志记录

### 输入
- 分类配置
- 时间窗口
- 抓取计划

### 输出
- 原始论文候选列表

---

## 6.2 数据清洗与去重模块

### 功能描述
对抓取到的论文进行标准化处理、版本归并和重复过滤。

### 功能要求
1. 标准化标题、作者、摘要、发布时间等字段
2. 识别同一论文不同版本
3. 基于 arXiv ID / DOI / 标题相似度去重
4. 标记新论文 / 更新论文 / 被过滤论文
5. 输出可入库的标准论文记录

### MVP 去重规则
按以下优先级判断：
1. DOI 完全一致
2. arXiv ID 完全一致
3. 标题高相似度 + 作者重叠
4. 标题中版本号差异归并

---

## 6.3 标签与归类模块

### 功能描述
为论文自动打上分类、主题、方法、任务和领域标签。

### 功能要求
1. 每篇论文至少绑定 1 个一级分类
2. 每篇论文至少映射到 1 个主题
3. 支持多标签
4. 支持标签置信度
5. 支持人工覆盖规则扩展

### 标签类型
- category
- topic
- method
- task
- domain
- application

---

## 6.4 排序与评分模块

### 功能描述
对新增论文进行评分排序，并确定进入日报、观察池和突破池的候选。

### 评分维度
- relevance
- novelty
- credibility
- impact
- readability
- freshness

### 默认权重
- relevance：0.35
- novelty：0.20
- credibility：0.15
- impact：0.15
- readability：0.10
- freshness：0.05

### 功能要求
1. 支持不同分类独立排序
2. 支持用户偏好加权
3. 支持突破候选标记
4. 输出排序理由

---

## 6.5 知识库模块

### 功能描述
将论文、版本、切片、分析结果和知识链接统一入库，供报告生成与对话问答使用。

### 功能要求
1. 存储论文主记录
2. 存储版本记录
3. 存储全文切片
4. 存储向量索引
5. 存储分析结果
6. 存储知识链接
7. 支持按分类 / 主题 / 时间 / 关键词检索
8. 支持历史报告和单篇论文详情查询

---

## 6.6 报告生成模块

## 6.6.1 日报

### 功能描述
按分类输出每日重点论文。

### 功能要求
1. 按分类分段展示
2. 每个分类最多 5 篇
3. 若高质量论文不足，允许少于 5 篇
4. 每篇至少包含：
   - 标题
   - 一句话结论
   - 要点
   - 分析
   - 观点
   - 原文
   - 知识链接
5. 每个分类头部输出：
   - 今日新增数量
   - 入选数量
   - 今日趋势判断

---

## 6.6.2 周报

### 功能描述
聚合最近 7 天新增内容并归纳趋势。

### 功能要求
1. 输出各分类高频主题
2. 输出本周代表性论文
3. 输出本周趋势变化
4. 输出下周建议观察方向
5. 支持纳入突破候选汇总

---

## 6.6.3 专题观察

### 功能描述
围绕指定主题生成阶段性研究观察报告。

### 功能要求
1. 支持配置主题范围
2. 支持配置时间窗口
3. 输出主题背景
4. 输出关键论文
5. 输出方法路线与演进
6. 输出当前瓶颈和后续观察点
7. 输出推荐阅读路径

---

## 6.6.4 突破追踪

### 功能描述
识别具有潜在突破意义的论文或方向，并持续追踪其演进。

### 功能要求
1. 建立突破候选池
2. 支持自动规则筛选
3. 支持人工确认
4. 输出突破理由
5. 支持在日报 / 周报 / 专题中引用
6. 支持对同一方向持续更新追踪状态

---

## 6.7 对话式研究助手

### 功能描述
用户可基于日报、知识库和主题内容进行进一步追问。

### 支持场景
1. 单篇论文精读
2. 多篇论文比较
3. 背景知识补充
4. 阅读路径构建
5. 主题趋势追问
6. 突破判断追问

### 功能要求
1. 回答需基于知识库和论文上下文
2. 支持调用论文全文关键段落
3. 支持返回相关论文与知识链接
4. 支持继续追问链路

---

## 6.8 知识链接模块

### 功能描述
为每篇重点论文生成前置、相似、竞争、后续和跨域等知识链接。

### 链接类型
- prerequisite
- related
- competing
- followup
- cross_domain

### 功能要求
1. 每篇重点论文至少生成 2 条知识链接
2. 每条链接需输出原因说明
3. 支持在报告和对话中展示

---

## 6.9 输出与推送模块

### 输出形态
- Markdown
- HTML
- Web 页面
- 飞书消息
- 企业微信消息

### 功能要求
1. 支持定时推送
2. 消息端仅发送摘要 + 链接
3. 长文输出由 Markdown / HTML / Web 承载
4. 支持失败重试
5. 支持历史报告留存和回查

---

## 6.10 泛读与精读模式（新增）

### 功能描述
用户点击具体论文卡片后，提供“泛读”和“精读”两种模式，辅助用户理解论文核心内容，并支持选择性入库。

### 功能要求
1. **泛读模式**：
   - 调用大语言模型，以中文形式输出简要的分析。
   - 介绍论文的主旨要义、核心创新点和结论。
2. **精读模式**：
   - 提供文章的全文翻译。
   - 提供对话式助手，辅助用户理解和学习论文细节。
3. **入库操作**：
   - 泛读和精读产生的内容（分析、翻译、对话记录）均支持用户选择性地加入知识库。

---

## 6.11 知识网络可视化（新增）

### 功能描述
将加入知识库中的知识点进行可视化展示，形成知识点网络，帮助用户直观理解知识之间的关联。

### 功能要求
1. **可视化展示**：以网络图的形式展示知识点及其关联。
2. **节点交互**：
   - 点击具体的知识点节点，侧边栏或弹窗展示相应的详细知识内容。
   - 提供穿透功能，允许用户从知识点直接跳转到原文的出处段落或页面。

---

## 6.12 闪念胶囊增强（新增）

### 功能描述
扩展现有的闪念胶囊功能，支持更丰富的内容录入形式。

### 功能要求
1. **多格式支持**：除了现有的粘贴文字，增加对上传文档（如 PDF, Word）的支持。
2. **多模态支持**：支持图片、音频等模态的数据录入和解析。

---

## 6.13 前端展示优化（新增）

### 功能描述
优化 Web 端的信息展示方式，提高用户的阅读效率和体验。

### 功能要求
1. **信息分栏展示**：将从 arXiv 和 GitHub 获取的信息在界面上进行明确的分栏或分块展示。
2. **论文卡片优化**：
   - 卡片内容必须展示：标题、摘要、关键字。
   - 关键字必须以醒目的标签（Tag）形式展示，便于快速扫视。

- 配置驱动
- 支持任务重跑
- 日志清晰
- 节点级状态可追踪

### 7.4 可扩展性
