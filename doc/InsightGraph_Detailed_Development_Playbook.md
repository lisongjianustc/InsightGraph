# InsightGraph 详细开发方案与执行手册 (n8n + Dify 架构版)

## 1. 文档目的

这份文档用于指导基于 **n8n + FastAPI + Dify + Vue3** 全新架构的 InsightGraph 项目开发。
取代了早期基于 LangGraph 和飞书推送的方案，完全聚焦于**本地化、Web 端闭环**的开发路径。

它解决三个核心问题：
1. 明确当前已完成的 Stage 1 进度基线。
2. 拆解 Stage 2 (阅读体验与 AI 赋能) 和 Stage 3 (图谱可视化) 的具体开发步骤。
3. 为接下来的每一次 AI 辅助编程提供清晰的上下文与 Prompt 边界。

---

## 2. 默认实施假设与边界

### 2.1 产品范围
- **闭环管理**：不再接入飞书/企业微信，所有操作、展示、通知均在 Vue3 前端完成。
- **抓取依赖**：完全依赖 n8n 的定时 Workflow 进行抓取和初步清洗，后端仅暴露接收 Webhook。
- **AI 与知识库引擎**：完全依赖本地部署的 Dify（提供 Dataset API 与 Chat API）。
- **图谱渲染**：初期采用关系型数据库 (PostgreSQL) 存储节点和边，前端用 ECharts/D3 渲染，不引入图数据库 (Neo4j) 以降低维护成本。

### 2.2 工程栈默认值
- **数据抓取**：n8n
- **后端**：Python 3.11 + FastAPI + SQLAlchemy + PostgreSQL
- **AI 层**：Dify
- **前端**：Vue 3 + Vite + TailwindCSS + Element Plus

## 4. 当前开发基线 (Stage 1 已完成)

根据《交接文档》，以下内容无需再次开发：
1. **环境层**：PostgreSQL, Redis, n8n, FastAPI (端口 8000), Dify (端口 5001) 的 Docker 编排已全部打通。
2. **抓取层**：n8n 已经配置好了 GitHub 与 Arxiv 的每日定时抓取并推送至 FastAPI `POST /api/feed/incoming`。
3. **数据层**：FastAPI 已创建 `FeedItem` 表。
4. **AI 层**：FastAPI 封装了 `dify_service.py` 实现了向 Dify Dataset 推送文本的功能。
5. **前端层**：Vue3 已搭建了基础侧边栏，并能从 `/api/feed/list` 获取数据渲染简单列表。

## 5. Stage 2 详细执行计划 (Web 阅读体验与 AI 赋能)

**目标**：把当前的“简陋列表”升级为“具备高级阅读辅助和整理能力的知识看板”。

### 5.1 子阶段 2.1：前端分栏与卡片展示优化
**负责人**：AI 辅助编码
**核心任务**：
- 重构 `FeedView.vue`，将数据源（arXiv 和 GitHub）在界面上进行分栏（或分 Tab）展示。
- 优化列表卡片组件，提取出核心字段：标题、摘要。
- 将 `keywords` 字段解析为数组，使用带颜色的 `Tag`（如 Element Plus 的 `<el-tag>`）在卡片上醒目展示。

### 5.2 子阶段 2.2：泛读模式 (Skim Reading)
**负责人**：AI 辅助编码 + 你(Dify 工作流配置)
**核心任务**：
- **前端**：在卡片上增加“泛读”按钮，点击弹出弹窗或抽屉。
- **后端**：新增 `/api/reader/skim` 接口，调用 Dify 的大语言模型接口（或直接请求本地大模型），传入论文摘要/内容，要求返回中文的核心要点与主旨。
- **交互**：泛读内容下方提供“保存至知识库”按钮，允许用户将这篇泛读总结作为一条独立知识录入 Dify Dataset。

### 5.3 子阶段 2.3：精读模式 (Deep Reading)
**负责人**：AI 辅助编码 + 你(Dify 聊天应用配置)
**核心任务**：
- **前端**：增加“精读”按钮，点击进入全屏/宽幅视图。
- **排版**：左侧展示全文或长摘要的双语对照翻译；右侧内嵌一个 Chatbot 聊天框。
- **后端交互**：左侧翻译可调用特定大模型 API；右侧聊天框通过后端转发，调用 Dify 中配置好的 Chatbot API 进行基于该文献上下文的 QA 问答。
- **入库交互**：允许将有价值的 QA 对话一键沉淀到知识库中。

### 5.4 子阶段 2.4：多模态闪念胶囊增强
**负责人**：AI 辅助编码
**核心任务**：
- **前端**：升级原有的“闪念胶囊”纯文本输入框，接入上传组件（支持拖拽上传）。
- **后端**：新增 `/api/capsule/upload` 接口，接收文件（PDF、图片、Word 等）。
- **解析逻辑**：后端集成基础的文件读取库（如 `PyPDF2` 或简单 OCR 库），提取文本内容后落入数据库并推送到 Dify。

## 6. Stage 3 详细执行计划 (知识网络可视化)

**目标**：把离散存入 Dify 知识库的内容，在界面上串联成网。

### 6.1 子阶段 3.1：关系数据结构建立
- **后端**：在 PostgreSQL 中建立 `GraphNode` 和 `GraphEdge` 表。
- 当用户将论文、泛读笔记、闪念胶囊入库时，后端不仅调用 Dify，同时在 Node 表写入节点信息。
- 如果泛读笔记关联到特定论文，自动在 Edge 表写入 `(笔记节点) -> relates_to -> (论文节点)` 的连线数据。

### 6.2 子阶段 3.2：可视化图谱渲染
- **前端**：引入 `ECharts` 的 Graph 模块或 `D3.js`。
- **后端**：提供 `/api/graph/data` 接口返回节点和连线的 JSON 数据。
- **交互**：前端渲染力导向图。用户点击某个节点时，侧边栏滑出该节点的详细内容（如果是论文，展示论文摘要；如果是胶囊，展示胶囊内容），并提供“查看原文”跳转按钮。

## 7. 给 AI 的分步开发 Prompt 指南

在实际开发中，请按顺序使用以下 Prompt 交由 AI（如 Codex/Claude）执行，避免一次性输入过多需求。

### Prompt 2.1：前端信息分栏与卡片优化
```text
请帮我重构 Vue3 前端的 FeedView.vue 页面。
当前状态：已经可以从后端 /api/feed/list 获取数据并简单渲染。
需求：
1. 实现双栏布局或双 Tab 切换，将 `source` 为 `arxiv` 的数据和 `github` 的数据分开展示。
2. 优化论文/项目的卡片组件，卡片必须清晰展示标题、摘要。
3. 数据中的 `keywords` 字段（逗号分隔或 JSON 数组），需要被解析并使用带颜色的 Tag 标签展示在卡片上。
请直接修改 Vue 组件代码，保持与 TailwindCSS / Element Plus 的样式兼容。
```

### Prompt 2.2：泛读模式与 Dify 接入
```text
接下来我们实现“泛读模式”。
需求：
1. 后端 (FastAPI)：新增一个接口 `POST /api/reader/skim`，接收 `feed_id`，利用之前写好的 Dify 接口调用能力（或直接用 LLM），生成中文的“主旨要义和核心结论”分析。
2. 前端 (Vue3)：在论文卡片上添加“泛读模式”按钮。点击后打开抽屉或弹窗，请求后端获取分析并展示。
3. 在弹窗底部提供“选择性入库”按钮，点击后将这段分析文字和原文链接一起，调用现有的 `save_to_kb` 逻辑推送到 Dify 知识库中。
请给出后端路由代码和前端弹窗组件代码。
```

### Prompt 2.3：精读模式（翻译与对话对话）
```text
请帮我实现“精读模式”的框架。
需求：
1. 前端新增全屏的“精读视图”页面。左侧是论文的详细内容（预留全文对照翻译的位置），右侧是一个类似于 ChatGPT 的对话聊天框。
2. 后端新增 `/api/reader/chat` 接口，接收用户的提问和当前论文的上下文，转发给 Dify 的 Chatbot API，然后将流式或阻塞响应返回给前端。
3. 允许用户在右侧对话框中，勾选有价值的问答记录，点击“入库”沉淀为知识点。
请先提供前后端的接口定义和基础 UI 结构代码。
```

### Prompt 2.4：多模态闪念胶囊增强
```text
我们需要对闪念胶囊功能进行增强。
需求：
1. 前端：在现有的文字输入框旁，增加一个文件上传区域（支持拖拽），允许上传 PDF、Word、图片等文件。
2. 后端 (FastAPI)：新增 `/api/capsule/upload` 接口接收上传的文件。使用 Python 相关库（如 PyPDF2 提取 PDF，或 Python-docx 提取 Word）提取文本内容。
3. 提取出文本后，将其作为一条新的闪念胶囊记录落库，并推送到 Dify 知识库中。
请给出前后端代码实现。
```

### Prompt 3.1：知识网络可视化
```text
我们现在开始实现 Stage 3 的知识图谱可视化。
需求：
1. 后端：使用 SQLAlchemy 建立简单的 GraphNode 和 GraphEdge 表，并编写一个 `/api/graph/data` 接口，返回包含 nodes 和 edges 数组的 JSON 数据。
2. 前端：新建一个 GraphView.vue 页面，引入 ECharts (力导向图 Graph) 渲染后端返回的节点数据。
3. 交互：配置 ECharts 的点击事件，当点击某个节点时，弹出一个侧边栏展示该节点的具体知识内容，并提供一个穿透到原文的链接。
请提供后端的模型定义、接口逻辑，以及前端可视化的核心代码。
```

## 8. 注意事项
1. **不要脱离实际进度**：当前已完成环境部署，不要再让 AI 生成基础的 Docker 配置文件，直接进入业务逻辑。
2. **接口规范**：前后端联调时，确保 Vite 配置了对 `http://localhost:8000` 的 Proxy，前端直接请求 `/api/...`。
3. **LLM 调试边界**：对于泛读/精读模式，AI 重点负责接口连通和前端 UI 搭建；具体大模型的提示词（Prompt）质量，需由你在 Dify 后台中自行调整优化。
