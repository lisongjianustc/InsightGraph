# InsightGraph v1.1.0 迭代开发计划 (Iteration Plan)

基于对系统的现状分析与用户价值的考量，我们将上述 5 个迭代点按照**“修复痛点 -> 夯实基础 -> 扩展能力 -> 提升体验”**的逻辑，划分为三个迭代阶段。

## 🎯 优先级建议与阶段划分

### Phase 1: 体验修复与基础加固 (优先级: 高)
*   **目标**：解决目前最影响日常使用的两个痛点：偶发的网络失败无法重试，以及历史胶囊难以回溯管理。
*   **包含迭代**：
    *   **[迭代 1] 今日资讯的局部与全局手动刷新**：提供重试机制，挽救大模型超时或网络波动造成的“死卡片”。
    *   **[迭代 4] 闪念胶囊的历史记录查看及管理**：将单薄的胶囊列表升级为具备搜索、分页、编辑、删除功能的真正的“笔记库”。

### Phase 2: 信息源扩展与被动温故 (优先级: 中)
*   **目标**：让系统能够接入更广泛的知识源，并建立长效的知识留存机制。
*   **包含迭代**：
    *   **[迭代 5] 动态信息源配置**：将硬编码的 arXiv 抓取逻辑重构为基于数据库配置的动态订阅中心（RSS/Atom），支持用户零代码添加任意博客或期刊源，并在前端自动生成分类页签。
    *   **[迭代 3] 每日温故小卡片**：在资讯流顶部插入复习模块，每日从精读库或胶囊库中随机/智能提取一条历史知识，实现间隔重复（Spaced Repetition）。

### Phase 3: 主动探索与深度检索 (优先级: 中低)
*   **目标**：从被动的“订阅-阅读”，走向主动的“搜索-归档”。
*   **包含迭代**：
    *   **[迭代 2] 主动条件搜索与归档**：新增独立的检索模块，允许用户直接向外部学术 API（如 arXiv）发起条件搜索，并将选中的结果精准入库，且与现有的自动抓取库实现完美的去重与整合。

## 🛠️ 具体实施方案 (Implementation Details)

### [Phase 1]
**1. 资讯刷新重试机制**
*   **后端**：
    *   在 `main.py` 新增 `POST /api/feed/{feed_id}/retry`。
    *   逻辑：清除 `error_message` 和状态位，重新将此记录压入 `BackgroundTasks`，触发 `dify_client.generate_skim_summary` 或相应的处理函数。
*   **前端**：
    *   在 `FeedView.vue` 顶栏增加全局的「立即同步」按钮（调用现有的 `/api/settings/sync` 并优化反馈状态）。
    *   在卡片组件中，当检测到失败状态（如大模型超时报错），显示带有 `RefreshRight` 图标的「重新生成」按钮，点击调用重试接口。

**2. 胶囊库高级管理**
*   **后端**：
    *   修改 `GET /api/capsules`：支持 `skip` 和 `limit` 分页，支持 `?keyword=` 模糊匹配（搜索 `title` 或 `content`）。
    *   新增 `PUT /api/capsules/{id}`（更新内容与图谱节点）和 `DELETE /api/capsules/{id}`（删除记录与图谱边）。
*   **前端**：
    *   在 `CapsuleView.vue` 增加顶部搜索栏与底部 Pagination。
    *   为每条胶囊增加操作下拉菜单（编辑/删除）。
    *   引入弹窗或行内编辑模式处理更新。

### [Phase 2]
**1. 动态信息源引擎**
*   **数据库**：
    *   新建 SQLAlchemy 模型 `SourceConfig` (`id`, `name`, `type`, `url`, `is_active`)。
*   **后端**：
    *   重构 `rss_service.py`：读取 `SourceConfig` 中所有 `is_active=True` 的记录进行循环抓取。
    *   新增 `/api/settings/sources` 的 CRUD 接口。
*   **前端**：
    *   在 `SettingsView.vue` 新增“信息源管理”表格，支持增删改查 RSS 链接。
    *   在 `FeedView.vue` 的顶栏，动态请求活跃的 source 列表，生成 `[全部] [arXiv] [自定义源 A]...` 的过滤标签组。

**2. 每日温故卡片**
*   **后端**：
    *   新增 `GET /api/review/daily` 接口。
    *   逻辑：每天凌晨（或首次请求时），从 `Capsule` 或状态为精读过的 `FeedItem` 中，使用 `ORDER BY RANDOM() LIMIT 1` 抽取一条。为保证当天不变，可存入 Redis 缓存或内存变量，设置 24h 过期。
*   **前端**：
    *   在 `FeedView.vue` 列表正上方渲染一个样式独特的卡片（如带有灯泡图标和渐变背景）。
    *   展示温故内容，点击可直接跳转图谱或详情。

### [Phase 3]
**1. 外部检索与精准入库**
*   **后端**：
    *   新增 `/api/search/external` 接口，接收关键词、作者、日期等参数。
    *   封装对外部 API（例如 `export.arxiv.org/api/query` 或未来可能的其他聚合搜索 API）的请求，解析为统一的 JSON 结构返回。
    *   新增 `/api/search/import` 接口，接收外部文献对象。
    *   入库前根据 `url` 或 DOI 检查 `FeedItem` 是否已存在，若存在则跳过并标记，否则作为新记录入库并触发后续泛读流程。
*   **前端**：
    *   新增独立路由 `/search` (SearchView.vue)。
    *   设计检索表单与搜索结果列表。
    *   结果列表右侧提供复选框，支持批量「添加到知识库」。添加成功的项目置灰或显示“已入库”标签。
